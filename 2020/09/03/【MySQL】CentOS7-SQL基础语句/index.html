MySQL基础语法之SQL基础

SQL介绍

  • SQL 指结构化查询语言

  • SQL 使我们有能力访问数据库

  • SQL 是一种 ANSI 的标准计算机语言，是关系型数据库中通用的一类语言

SQL常用类型

  • mysql客户端自带的语句 用help指令查看

  • Server端命令，用help content指令查看，常用的Server端命令有：

    • DDL: 数据定义语言
    • DCL: 数据控制语言
    • DML: 数据操作语言
    • DQL: 数据查询语言

SQL的各种名词

sql_mode/SQL模式: 作用是规范SQL语句书写方式。

    通过 “select @@sql_mode;” 查看现有的SQL_mode

    通过修改脚本，可以设置数据库的SQL_mode

字符集（charset）及校对规则（collation）

    • 字符集：常用的字符集有：utf8, utf8mb4（推荐）两类。utf8mb4支持编码比utf8更多，比如emoji字符utf8就不支持。因为utf8最大存储长度为3字节（utf8mb4为4字节），而一个emoji字符 占4字节。
    • 校对规则：每种字符集有多重校对规则（排序规则），校对规则影响到排序的操作，比如大小写是否敏感。

数据类型(data type):

  1. 数字类型：

  2. 字符串类型：

注意： char(n) 和 varchar(n) 中括号中 n 代表字符的个数，并不代表字节个数，比如 CHAR(30) 就可以存储 30 个字符。

CHAR 和 VARCHAR 类型类似，但它们保存和检索的方式不同。它们的最大长度和是否尾部空格被保留等方面也不同。在存储或检索过程中不进行大小写转换。

BINARY 和 VARBINARY 类似于 CHAR 和 VARCHAR，不同的是它们包含二进制字符串而不要非二进制字符串。也就是说，它们包含字节字符串而不是字符字符串。这说明它们没有字符集，并且排序和比较基于列值字节的数值值。

BLOB 是一个二进制大对象，可以容纳可变数量的数据。有 4 种 BLOB 类型：TINYBLOB、BLOB、MEDIUMBLOB 和 LONGBLOB。它们区别在于可容纳存储范围不同。

有 4 种 TEXT 类型：TINYTEXT、TEXT、MEDIUMTEXT 和 LONGTEXT。对应的这 4 种 BLOB 类型，可存储的最大长度不同，可根据实际情况选择。

varchar(n)类型，在存储数据时会先判断字符长度，然后合理分配存储空间，而char立即分配空间，存不满的部分空格填充，固定长度序列中推荐char。

varchar(n)类型除了存储数据之外，还会消耗1-2bit空间用于存储字符长度。

enum(‘bj’,’sh’,’sd’,…) 枚举类型，是一种字符串类型，将所有可能的字符进行枚举，存储时可以存储相应的下标索引，提升检索性能，节省存储空间。

约束

  • Primary Key ： 主键约束，作用是唯一且非空，每张表只能有一个主键

  • not null： 非空约束，作用必须非空，建议每个列都设置非空

  • unique kye： 唯一约束，作用，必须是不重复的

  • unsigned: 约束数字列为非负数。

其它属性

  • default： 默认值

  • comment： 注释
