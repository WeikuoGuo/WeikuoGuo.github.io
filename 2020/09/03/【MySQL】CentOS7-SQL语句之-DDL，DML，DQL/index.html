DDL 数据定义语言

库定义

库名，库属性

  • 创建库：CREATE DATABASE 库名 CHARSET utf8mb4;

    1. 库名小写，与业务有关，不要数字开头，库名不要太长, 不要使用保留字段。
    1. 必须定制字符集。

  • 查询库: show databases; show create database 库名; (查询建库语句)

  • 修改库：alter database 库名 charset utf8;

    • 注意，修改后的字符集应当比修改前更大，是前者的严格超集，否则出现乱码。

  • 删除库：drop database 库名;

    • 注意：生产数据库中除了管理员没有任何人有删库权限。

表定义

  • 创建表：CREATE TABLE `库名`.`表名` (column_name column_type);
    • 表名小写，与业务有关，不要数字开头，名不要太长, 不要使用保留字段。
    • 必须设置存储引擎和字符集
    • 选择合适的数据类型： 合适，简短，足够
    • 必须要有主键
    • 每个列尽量设置not null, 不知道填什么设置默认值。
    • 每个列要有注释
    • 列名不要太长

  • 查询表：

    • show tables; 查看库里有哪些表
    • desc wp_users; 查看单个表的定义及其它设置
    • show create table wp_users; 查看建表语句

  • 修改表：

    • 添加列：alter table 表名 add column 列名 列属性;
    • 修改列：alter table 表名 modify 列名 列属性;(所有的属性都要重新写，不然会被修改后的覆盖)
    • 删除列：alter table 表名 drop 列名;

  • 删除表： drop table 表名;

线上DDL（alter）操作对于生产的影响

在MySQL中，DDL语句在对表进行操作时，是要锁“元数据表”的，此时所有修改类的命令无法正常运行。

所以在对于大表，业务繁忙的表，进行线上DDL操作时，要谨慎。尽量避开业务繁忙期间。或者借助线上工具如pt-osc(pt-online-schema-change)，pt-ost进行操作，减少锁表的影响。如果是8.0版本，可以不使用pt工具。

元数据类似于Linux中的Inode信息。

  • 扩展

    • 伪删除：并没有真正删除，只是在查询的时候查不到该数据行，可以进行回退。
    • delete from 表名；drop table 表名；truncate table 表名 三者的区别
      1. 以上三条命令都可以删除全表数据
      1. 区别：

    delete是逐行进行删除，数据行较多时，会非常慢。并没有真正从磁盘上删除，只是在储存层面打上标记，磁盘不立即释放。HWM高水位线不会降低。

    drop语句将表结构的元数据和数据行物理层次删除。

    truncate是清空表段中的所有数据页，磁盘空间释放，HWM高水位线会降低。
