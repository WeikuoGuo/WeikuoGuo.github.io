MySQL的逻辑结构（操作对象）

MySQL

库： 库名 + 库属性

表： 表名 + 表属性 + 表内容 + 列

MySQL的物理存储结构

  • 扇区：基本存储单元，表示一段连续的磁盘空间，顺序IO

  • block: 由Linux文件系统提供的存储单元，默认4kb，由连续的8个扇区组成。

  • 页（page）：MySQL最小存储单元，默认16kb，由4个连续的block组成

  • 区/簇（extent）：MySQL定义的存储单元，默认1Mb, 由64个连续的页组成。

  • 段：一个表就是一个段，包含一个或多个区

MySQL存储结构的意义

实现聚簇索引，利于SQL的优化，提升检索效率

MySQL基础管理

用户管理

  1. 用户的作用

登录MySQL数据库，管理MySQL对象（表）

  1. 用户的定义

用户名@’白名单’

白名单的意义：一个地址列表，允许在列表中的IO登录MySQL，管理MySQL

oldguo@’localhost’: oldguo用户能够通过本地登录MySQL（socker）
oldguo@’10.0.0.10’: oldguo用户只能通过10.0.0.10远程登录MySQL服务器
oldguo@’10.0.0.%’: oldguo用户只能通过10.0.0.xx/24远程登录MySQL服务器
oldguo@’10.0.0.5%’: oldguo用户只能通过10.0.0.50-59远程登录MySQL服务器
oldguo@’10.0.0.0/255.255.254.0’: oldguo用户只能通过255.255.254.0网关登录MySQL数据库服务器
oldguo@’%’: oldguo用户能通过所有IP登录。

MySQL连接管理

MySQL自带客户端

mysql:

参数列表：

参数名 含义
-u 用户名
-p 密码
-S 本地Socket文件位置
-h 数据库IP地址
-p 数据库端口号
-e 免交互执行数据库命令
< 导入SQL脚本

Socket连接：

前提：必须实现授权localhost登录

mysql -u用户名（默认root） -p密码 -S位置（默认/tmp/mysql.sock）

TCP/IP连接：

前提：必须提前创建好可以远程连接的用户

mysql -u用户名（默认root） -p密码 -h IP地址 -P 端口号（默认3306）

注意：可以通过show processlist;查看是通过哪种方式连接的。
